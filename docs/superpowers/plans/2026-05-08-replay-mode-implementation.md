# Replay Mode Implementation Plan

> **For agentic workers:** Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the Replay Mode — a manual, on-demand, watch-anchored controlled retro-execution of the bot's pipeline on past time windows. The user picks a window, steps candle by candle, observes the bot's reasoning (Detector → Reviewer → Finalizer → tracking → feedback), with all side effects neutralized (no Telegram, no writes to live tables, no automatic activity).

**Architecture:** Hexagonal (ports + adapters). New thin Temporal workflow `replaySessionWorkflow` (~150 lines) drives EXISTING activities (`runDetector`, `runReviewer`, `runFinalizer`, `runFeedbackAnalysis`, `persistEvent`, `markSetupClosed`, `notifyTelegram*`) via a single `replayContext` parameter that branches their DI to replay-scoped adapters. No parallel orchestrator, no duplication of state machine / scoring / tracking. The bot replayed is rigorously the bot live, modulo neutralized side effects.

**Tech Stack:** Bun · TypeScript strict · Drizzle ORM · Postgres 16 · Temporal · Zod · Handlebars · `@anthropic-ai/claude-agent-sdk` + OpenRouter · grammy (Telegram) · Biome · `bun test` · testcontainers · lightweight-charts v5 · React + react-router-dom + tanstack-query.

**Spec reference:** `docs/superpowers/specs/2026-05-08-replay-mode-design.md` (1185 lines).

**Golden rule:** Nothing runs in replay unless the user explicitly clicks. No cron, no priceMonitor, no trackingLoop, no Temporal sleeps, no child workflows, no polling. Idle until signaled. If user closes the tab, zero LLM cost is incurred.

## Conventions used in this plan

- **TDD-friendly tasks** : when reasonable, write the test signature first as a contract, then the implementation. The test text is the spec for that function.
- **Reuse over recreate** : every new piece must justify why a live equivalent can't be parametrized. Default answer should be "we extend the live one with a `replayContext?: ReplayContext` arg".
- **One commit per task** unless the task is split into preparatory steps. Each task ends with a ready-to-use commit message.
- **Spec back-references** : `(spec §N)` points to the corresponding section in `2026-05-08-replay-mode-design.md`.
- **Test commands** : at the end of each task, the exact `bun test ...` command to verify it.
- **Golden rule check** : tasks that modify the workflow path include a checkbox "verifies no automatic activity fires without a signal" — this is non-negotiable.

**Why** (spec §12 Jalon 1) : à la création d'une session, on copie les events live de la fenêtre dans `replay_events` pour fournir une baseline visible. Le Jalon 2 ajoutera des events PAR-DESSUS via le step.

- Create `test/domain/replay/copyLiveEvents.test.ts`

**Contract** :
```ts
export async function copyLiveEventsToReplay(deps: {
  liveEventStore: EventStore;
  replayEventStore: ReplayEventStore;
  setupRepository: SetupRepository;
}, args: {
  sessionId: string;
  watchId: string;
  windowStartAt: Date;
  windowEndAt: Date;
}): Promise<{ copied: number }>;
```

Behavior :
1. List setups of the watch whose `created_at` is in `[windowStart, windowEnd]` OR whose lifecycle overlaps the window.
2. For each, list its events with `occurred_at` in the window.
3. Insert each event into `replay_events` with the session_id, preserving sequence order via `append`.
4. Return count.

**Tests (with InMemory fakes)** :
- [ ] No setups in window → 0 copied
- [ ] 1 setup with 5 events in window → 5 copied, sequence 1..5
- [ ] 2 setups in window → both copied, sequence interleaved by occurred_at
- [ ] Events outside window excluded

```ts
export class CachedLLMProvider implements LLMProvider {
  constructor(

    private cache: LLMResponseCacheStore,

  async call(input: LLMInput): Promise<LLMOutput & { fromCache: boolean }> {
    const hash = computeInputHash(input);   // reuses existing inputHash domain function
    const cached = await this.cache.get(hash);
    if (cached) {

    return { ...out, fromCache: false };
```

```ts
export async function replaySessionWorkflow(args: ReplaySessionArgs): Promise<void> {
  const state: ReplayWorkflowState = initState(args);

  setHandler(replayTickSignal, async ({ tickAt }) => {
    if (state.status !== "READY") return;
    const t = new Date(tickAt);
    if (t > args.windowEnd) return;

    // Cost cap pre-check
    if (state.costSoFar >= args.costCap) {
      state.status = "COST_CAPPED";
      await persistReplayMeta("cost_capped");
      return;
    }

    const replayCtx: ReplayContext = {
      sessionId: args.sessionId,
      tickAt: t,
      lessonsMode: args.lessonsMode,
      feedbackMode: args.feedbackMode,
    };

    // 1. Detector
    const det = await runDetector({ ..., replayContext: replayCtx });
    state.costSoFar += det.costUsd;
    applyDetectorResultToState(state, det, t, args.configSnapshot);

    // 2. For each alive in REVIEWING / FINALIZING → Reviewer / Finalizer
    for (const setup of state.alive) {
      if (state.costSoFar >= args.costCap) break;
      if (setup.status === "REVIEWING") {
        const rev = await runReviewer({ ..., replayContext: replayCtx });
        state.costSoFar += rev.costUsd;
        applyVerdict(setup, rev);          // existing pure function from domain
      }
      if (setup.currentScore >= args.configSnapshot.scoreThresholdFinalizer && setup.status !== "TRACKING") {
        const final = await runFinalizer({ ..., replayContext: replayCtx });
        state.costSoFar += final.costUsd;
        applyFinalizerResultToState(setup, final, t);
      }
    }

    // 3. Tracking deterministic (no LLM)
    const candle = await fetchCandle({ symbol: args.watchSymbol, at: t });
    for (const setup of state.alive) {
      if (setup.status === "TRACKING") {
        applyIntraCandleTracking(setup, candle, replayCtx);  // SL prioritaire si conflict
      }
    }

    // 4. TTL check
    for (const setup of state.alive) {
      if (t >= setup.ttlExpiresAt && !isTerminal(setup.status)) {
        await persistEvent({ /* Expired */, replayContext: replayCtx });
        setup.status = "EXPIRED";
      }
    }

    // 5. Feedback loop on close
    if (args.feedbackMode === "run") {
      for (const setup of state.alive) {
        if (justClosedEligibleForFeedback(setup)) {
          await runFeedbackAnalysis({ ..., replayContext: replayCtx });
          state.costSoFar += feedbackCost;
        }
      }
    }

    // Move closed setups out of alive
    state.closedSetupIds.push(...alive.filter(isTerminal).map((s) => s.setupId));
    state.alive = state.alive.filter((s) => !isTerminal(s.status));

    state.playheadAt = t;
    if (t >= args.windowEnd) state.status = "COMPLETED";
  });

  setHandler(replayPauseSignal, () => { state.status = "PAUSED"; });
  setHandler(replayResumeSignal, () => { if (state.status === "PAUSED") state.status = "READY"; });
  setHandler(getStateQuery, () => state);

  await condition(() =>
    state.status === "COMPLETED" || state.status === "COST_CAPPED" || state.status === "FAILED"
  );
}
```

- [ ] Workflow started, idle, no activity called (verify activity stubs receive 0 invocations)
